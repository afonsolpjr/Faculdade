Podemos tentar dar uma certa propriedade a uma função usando raciocinio equacional.


Desafio: uma implementação mais eficiente de reverse:

reverse :: [a] -> [a]
reverse []      = []
reverse (x:xs)  = reverse xs ++ [x]

-- é util termos a definicao de ++

(++) :: [a] -> [a] -> [a]
[] ++ ys      = ys
(x:xs) ++ ys  = x:(xs ++ ys)


Desafio: queremos uma função
rev' :: [a] -> [a] -> [a]
que cumpra a propriedade
rev' xs ys = (rev xs) ++ ys 
-- a idéia do ys é um acumulador, que vai acumulando o que ja foi revertido, eu acho

-- como derivar ou tentar criar uma implementação de rev'?
-- estrategia : fingir que ja temos a funcao e tentar provar que a funcao já satisfaz a propriedade. assim, em algum momento iremos nos deparar com um passo faltante da prova.
a prova será por indução em dois casos:

caso []
rev' [] ys = (rev []) ++ ys

caso (x:xs)
rev' (x:xs) = (rev (x:xs)) ++ ys

# provando o caso []

vamos ir substituindo a equacao do lado direito

rev' [] ys  | (rev []) ++ ys

--usando a definicao de rev:

rev' [] ys  | [] ++ ys

-- usando a definicao de (++):

rev' [] ys  | ys

-- AQUI entra a mágica desta forma de derivar funções.... de acordo com esse raciocinio equacional aí, para concluirmos a nossa prova que não seria muito eficiente definir que rev' [] ys é igual a ys ? e é isso que fazemos!

rev' [] ys = ys


# hipotese de inducao
para qualquer (x:xs), se rev' xs ys = (rev xs) ++ ys, então rev' (x:xs) ys = (rev (x:xs)) ++ ys

# caso (x:xs)

rev' (x:xs) ys = (rev (x:xs)) ++ ys

-- usando definicao de rev

rev' (x:xs) ys = (rev xs ++ [x]) ++ ys

-- usando associatividade entre [x] e ys

rev' (x:xs) ys = (rev xs) ++ ([x] ++ ys)

-- usando a definicao de ++ na expresssao da direita

rev' (x:xs) ys = (rev xs) ++ (x:([]++ys)) = (rev xs) ++ (x:ys)

-- e aqui, do lado direito da equacao, já temos um caso para usar a hipotese de inducao, que dira que:
-- (rev xs) ++ (x:ys) = rev' xs (x:ys)

rev' (x:xs) ys = rev' xs (x:ys)
-- e tá definido bixo.
